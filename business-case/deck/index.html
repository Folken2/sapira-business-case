05 / 08 Data & reconciliation

Typed BOM in. Confident SAP code out.

# Per BOM line · Pydantic
class LineItem:
  line_ref         : str
  raw_description  : str
  grade            : str
  form             : Form | None     # sheet|plate|coil|tube…
  thickness_mm     : float | None
  quantity         : float
  uom              : UoM | None      # TON|KG|M
  notes            : str             # margin notes

# Extraction wraps lines with email_type
# (NEW|REVISION|DUP), project, bom_revision.
# ReconciledLine adds sap_code, confidence,
# status (auto|hitl), hitl_reason.

Friction #2 — text → SAP code · learns over time

1
Ground-truth map · built from history
60 days of past email → PO pairs gives us thousands of "Galv Steel B → AC-GALV-TB-001" mappings on day one.
2
Reconciler agent reasons · for new text
Reads candidate SAP codes, applies steel-domain rules (grade family, thickness, tolerance), scores its own confidence.
!
Below threshold → HITL
PO blocked until queue empty. By design.
↻
Every HITL decision feeds the map
The agent learns. Same uncertainty doesn't appear twice — accuracy compounds week over week.
