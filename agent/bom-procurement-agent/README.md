# bom-procurement-agent

> Generated by [meta-agent](https://github.com/Folken2/meta-agent)

## Quick Start

```bash
# Setup
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Configure
cp .env.example .env
# Edit .env — add your OPENROUTER_API_KEY

# Run
DEV_MODE=true python run_adk.py

# Or use the ADK web UI
DEV_MODE=true adk web .
```

The agent runs at `http://localhost:8000`. Use `/run_sse/` for streaming or the ADK web UI for interactive testing.

## Customization

### Adding Tools

Create a tool function in `bom_procurement_agent/tools/` and register it in `get_tools()`:

```python
from google.adk.tools import FunctionTool
from google.adk.tools.tool_context import ToolContext

def my_tool(param: str, tool_context: ToolContext) -> dict:
    """Description the LLM reads to decide when to call this tool."""
    return {"status": "success", "message": "result"}

my_tool_fn = FunctionTool(func=my_tool)
```

### Adding Skills

Create a subdirectory under `bom_procurement_agent/skills/` with a `SKILL.md` file:

```
bom_procurement_agent/skills/my-skill/
  SKILL.md           # Frontmatter (name, description) + instructions
  references/        # Optional: detailed reference files (loaded on demand)
```

Skills are auto-discovered at startup via `SkillToolset`.

### Adding Domain Knowledge

Place `.md` files in `bom_procurement_agent/contexts/`. They are injected into the system prompt automatically.

## Plugin Chain

This agent ships with 10 plugins — cross-cutting behaviors that apply to all interactions without touching agent code:

| Plugin | What it does |
|--------|-------------|
| **CostGuardPlugin** | Tracks USD cost per LLM call, enforces per-session budget |
| **TracePlugin** | Raw event JSONL + consolidated conversation JSON for evals |
| **ConsoleLoggerPlugin** | Color-coded terminal output |
| **ToolEventsPlugin** | Structured tool events for SSE streaming |
| **ContextFilterPlugin** | Keeps last N invocations in context window |
| **CachePlugin** | Session-scoped tool response caching with TTL |
| **ResiliencePlugin** | Circuit breaker + rate limiting for tools |
| **ReflectAndRetryToolPlugin** | Self-healing tool retry with LLM reflection |
| **SaveFilesAsArtifactsPlugin** | Saves user-uploaded files as session artifacts |
| **MemoryPlugin** | Markdown file-based long-term memory |

### Cost Tracking

Edit `bom_procurement_agent/plugins/pricing.json` to add or update model pricing (no code changes needed):

```json
{
  "moonshotai/kimi-k2.5": { "input": 0.0000005, "output": 0.000002 },
  "anthropic/claude-sonnet-4": { "input": 0.000003, "output": 0.000015 }
}
```

Prices are USD per token. The plugin auto-strips provider prefixes (`openrouter/model` matches `model`).

Set `COST_GUARD_BUDGET=0.50` in `.env` to cap spending at $0.50 per session.

## Deploy to Railway

This agent ships with `Dockerfile`, `railway.json`, and `.dockerignore` — ready to deploy.

### One-time setup

```bash
# Install the Railway CLI
brew install railway  # or: npm install -g @railway/cli

# Log in
railway login
```

### Deploy

```bash
# From the agent directory:
railway init         # Create a Railway project (select "Empty Project")
railway link         # Link this directory to the project (if not already)
railway up           # Build + deploy
```

### Configure environment variables

Set these in the Railway dashboard (or via `railway variables --set KEY=VALUE`):

| Variable | Required | Notes |
|----------|----------|-------|
| `OPENROUTER_API_KEY` | Yes | Your OpenRouter API key |
| `API_KEY` | Recommended | Enables Bearer-token auth on all endpoints |
| `DEV_MODE` | Optional | Set to `true` for in-memory sessions (no Postgres); defaults to `false` |
| `GOOGLE_API_KEY` | Optional | Required when `STREAMING_ENABLED=true` |
| `STREAMING_ENABLED` | Optional | Set to `true` for voice/video agents |

### Optional: Postgres for persistent sessions

By default the agent runs with in-memory sessions when `DEV_MODE=true`. For persistent sessions:

1. In the Railway dashboard, add a Postgres plugin to your project
2. Copy the Postgres `DATABASE_URL` from the plugin's Variables tab
3. Set `SESSION_SERVICE_URI=<that URL>` on your agent service
4. Set `DEV_MODE=false`

### Health check

Railway polls `/health` (configured in `railway.json`) to verify the container is ready before routing traffic. The `/health` endpoint is public and requires no auth.
